经验总结（更新中）

Created2020-08-19Updated2020-08-24

在这里总结一些踩过的坑以及一些小技巧。

  • 容易犯错的地方（fst原因大全）

    1. 数组大小有时候会是 2×1052\times 10^52×105 甚至是 5×1055\times 10^55×105，不要想当然直接开 maxn=1e5+9，运气好的话在pretest中RE了还能查出来，要是运气不好过了pretest那就等着fst吧
    2. 初始化的时候尽量写 maxx=-inf,minn=inf 这样的极端数据，无脑初始化为 0 是一个坏习惯
    3. 不开 long long 见祖宗，被坑了无数次了
    4. double 的精度并没有想象的那么高，用的时候要小心

  • 小优化

    1. 读01矩阵这样的东西用 scanf("%c",&x) 比用 scanf("%1d",&x) 稍微快一点

  • 一些经验

    1. 指针和宏定义稍有不慎就会出奇怪的毛病且非常难以调试，用不好就不要用
    2. 不要吃饱了撑着写一堆 register 变量或者位运算，有时候编译器自己会优化的，多用 register 并不一定使程序变快，甚至可能会负优化；多用位运算也不一定会变快，出错的概率倒是大大增加
    3. 想不出做法的时候试试暴力，有奇效
    4. 不只是序列可以分块做，图也是可以分块的
    5. 单调栈非常有用（被这东西坑了两次了）

  • 容易忘记的小知识点

    1. 斐波那契数列通项公式 fn=(1+52)n−(1−52)n5\large f_n=\dfrac{(\frac{1+\sqrt 5}{2})^n-(\frac{1-\sqrt 5}{2})^n}{\sqrt 5}fn​=5​(21+5​​)n−(21−5​​)n​

    2. 在只含加法和乘法的式子中，如果最后的运算结果需要对p取模，那么可以在运算过程中随便取模
